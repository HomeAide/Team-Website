Project Requirements

Addressing functional limitations

The user will need a way to input their functional limitations to the application.

Additionally, the application will also need to remember what these limitations are for purposes of personalization and to provide useful information to the user.

Identify difficult areas

Users will need to be able to identify trouble areas in their home

There will be a house diagram displayed to the user where they can select which rooms tend to give them the most difficulties. Like with functional limitations, the application will also remember what these rooms are to better assist the user with their needs.

Store user information

A database will be used to store user information

In this database will contain user limitations and unique profiles for each user. That way this information is stored somewhere in case the user switches devices.

Accessibility

The application needs to be accessible to a variety of physical limitations.

A colorblind mode or text-to-speech are examles of how the application can be made accessible. Regardless of whatever physical limitation a user is experiencing, the application will still be usable.

AT Recommendation

The application needs to be able to recommend Assistive Technology to a user if necessary

The application will use the users inputted functional limitations and difficult areas in their home to determine if the user may benefit from assistive technologies and what technologies would work best for their difficulties.

Database Accessibility

The client will need to be able to add, remove, or update information from the database.

The database will store two main types of information: user information and a list of AT devices. The client will need to be able to add, remove, or update user or AT device information within this database.
